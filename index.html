(Question 2): What are the differences between var, let and const?

var: It is function-scoped. It can be reassigned as well as redeclared.
let: Its scope is limited to a particular block. It can be reassigned but cannot be redeclared.
const: Its scope is also limited to a block and cannot be ressigned.

(Question 3): What is execution context?

Execution Context (EC) is an environment where JavaScript code is executed. It contains the variables, functions, and the scope in which the code is running.

(Question 4): What is creation phase and execution phase?

Creation Phase:

It happens before the code starts running. JavaScript scans the code and:
Creates the Global Object (window in browsers, global in Node.js).
Sets up the this keyword (depends on the execution context: window in the global scope).
Hoists function declarations and variables.

Execution Phase:

In this phase, JavaScript runs the code line by line.
Assigns values to variables.
Executes functions when they are invoked.

(Question 5): What is hoisting?

Hoisting is a JavaScript mechanism where variable and function declarations are moved to the top of their containing scope during the creation phase before code execution. Only the declaration is hoisted, not the initialization.

(Question 6): Difference between undefined vs not defined vs NaN?

Undefined:

A variable is declared but has not been assigned a value.

Not Defined:

A variable has not been declared in the current scope.

NaN:

A value that isn't a valid number. Eg: Math.sqrt(-1)

(Question 8): Explain Local Scope, Block Scope, Functional Scope and Scope Chain in javascript

Local Scope:

A variable declared inside a function is local to that function. It cannot be accessed outside the function.

Block Scope:

Introduced with let and const. Variables declared inside a block are only accessible within that block. var does not have block scope.

Function Scope:

Variables declared with var inside a function are accessible only within that function.

Scope Chain:

While accessing a variable, JavaScript searches in the current scope first. If not found, it moves up to the parent scope (outer function → global scope). This process is called the Scope Chain.
